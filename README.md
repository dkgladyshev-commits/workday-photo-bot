# Фотография рабочего дня — Telegram-бот (MVP)

Минимальный бот для фиксации того, на что реально уходит время за день, по трём
категориям: **рабочая задача**, **личная задача**, **перерыв**.

Без базы данных, без веб-интерфейса, без авторизации. Хранение — простой
`tasks.json`. Стек: Python + aiogram 3 + python-dotenv.

## Как это работает

1. Пишете задачу обычным сообщением (например, «Подготовить письмо клиенту»).
2. Выбираете категорию кнопкой — задача **сразу создаётся и стартует** (время пошло).
3. Если уже была активная задача, она **автоматически уходит на паузу**.
4. Управляете задачей кнопками: ⏸ Пауза / ▶️ Продолжить / ✅ Завершить.
5. `/today` показывает «фотографию дня» и работает как пульт: под сообщением —
   кнопки для всех активных и приостановленных задач (с именами), можно вернуться
   к любой paused-задаче.

## 1. Создать бота через BotFather

1. Откройте в Telegram [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot`.
3. Задайте имя бота и его username (должен заканчиваться на `bot`).
4. BotFather пришлёт **токен** вида `123456789:AAE...` — это `TELEGRAM_BOT_TOKEN`.

## 2. Получить и сохранить токен

Скопируйте `.env.example` в `.env` и вставьте токен:

```bash
cp .env.example .env
# затем откройте .env и подставьте свой токен
```

`.env`:

```
TELEGRAM_BOT_TOKEN=123456789:AAE...
```

## 3. Установить зависимости

Рекомендуется виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## 4. Запустить бота

```bash
python bot.py
```

В логах появится «Бот запущен. Ожидание сообщений…». Откройте бота в Telegram и
отправьте `/start`.

## 5. Команды

| Команда   | Действие                                                        |
|-----------|-----------------------------------------------------------------|
| `/start`  | Краткая инструкция                                              |
| `/today`  | Фотография дня + кнопки управления активными/паузными задачами  |
| `/active` | Показать текущую активную задачу                                |
| `/pause`  | Поставить активную задачу на паузу                              |
| `/stop`   | Завершить активную задачу                                       |
| `/clear`  | Очистить сегодняшний список (с подтверждением)                  |

Чтобы создать задачу — просто напишите её текст без слэша.

## Как считается время

Единый helper `effective_seconds(task, now)` — один источник правды:

- если задача **running**: `accumulated_seconds + (сейчас − last_started_at)`;
- иначе: `accumulated_seconds`.

Правила:

- В один момент у пользователя может быть только одна **running**-задача.
  Запуск новой/продолжение другой автоматически ставит текущую на паузу
  (с банковкой времени).
- При паузе и при завершении **running**-задачи остаток времени добавляется в
  `accumulated_seconds` один раз.
- Завершение **paused**-задачи время не меняет (защита от двойного счёта).
- Все даты считаются в таймзоне **Asia/Almaty** (timezone-aware), включая
  границы дня.

## Хранение

`tasks.json` создаётся автоматически при первом запуске:

```json
{
  "tasks": {
    "2026-06-14": [
      {
        "id": "uuid",
        "user_id": "123",
        "title": "Подготовить письмо клиенту",
        "category": "work",
        "status": "completed",
        "date": "2026-06-14",
        "created_at": "2026-06-14T10:00:00+05:00",
        "last_started_at": null,
        "completed_at": "2026-06-14T10:25:00+05:00",
        "accumulated_seconds": 1500
      }
    ]
  },
  "pending": {}
}
```

- Каждая задача привязана к `user_id` — несколько пользователей не видят и не
  трогают задачи друг друга.
- Запись атомарная (через временный файл) и защищена `asyncio.Lock`, поэтому
  быстрые нажатия кнопок не портят файл.

## Структура проекта

```
bot.py            — запуск, polling
handlers.py       — команды, текст -> pending, callback-кнопки, рендеринг
storage.py        — tasks.json, логика статусов и времени, asyncio.Lock
keyboards.py      — inline-клавиатуры
utils.py          — время (Almaty), format_duration, effective_seconds
test_logic.py     — оффлайн-проверка логики времени (python test_logic.py)
requirements.txt
.env.example
```

## Проверка логики без Telegram

```bash
python test_logic.py
```

Прогоняет ключевые сценарии: автопауза при создании новой задачи, продолжение
paused-задачи из списка, отсутствие двойного счёта при завершении, изоляция
пользователей.

## Границы MVP

Осознанно не реализовано: задачи через полночь учитываются по дате создания;
нет редактирования названия/категории после создания; нет дашборда и экспорта.
